## Зачем нужен Starshot

Встроенные инструменты Windows для скриншотов (Ножницы, Win+Shift+S) на HDR-дисплеях по-прежнему создают только 8-битные SDR-изображения — системный композитор сжимает 16-битный HDR-кадр, обрезая светлые участки и сужая цветовой охват. Распространённые сторонние инструменты (ShareX и др.) также ограничены традиционным конвейером GDI/BitBlt и не могут работать с HDR-данными.

Starshot напрямую получает исходный `R16G16B16A16Float` scRGB-фреймбуфер вывода дисплея с уровня DXGI, полностью сохраняя информацию о яркости HDR (до тысяч нит), и кодирует в 16-битный HDR AVIF или JPEG XL с метаданными цветового пространства BT.2020 + функция передачи PQ. При этом он предоставляет все функции, ожидаемые от универсального инструмента для скриншотов: автоматическое понижение для SDR-дисплеев, захват области, пакетное преобразование форматов и многое другое.

**Ключевые особенности**

- 🎯 **Полный конвейер HDR без потерь** — 16 бит на всём протяжении захвата, кодирования и управления цветом; без тональной компрессии с потерями
- 🧠 **Умное определение HDR/SDR** — Гистограмма maxCLL отличает настоящий HDR-контент от SDR-контента в HDR-обёртке
- ✂️ **Снимок области** — Оверлей с замороженным кадром на нескольких дисплеях, обнаружение окон + лупа для точного выбора
- 📋 **Нативный буфер обмена** — Нативное API Win32 напрямую записывает в буфер обмена, избегая проблем с отложенным рендерингом WinRT
- 🗂️ **Поддержка множества форматов** — AVIF / JPEG XL / UHDR JPEG / PNG, с инструментом пакетного преобразования
- 📦 **Портативный** — распакуйте и запускайте, права администратора не требуются

- Архитектура x64

## Возможности

### Конвейер HDR-захвата

Большинство инструментов для скриншотов даже на HDR-дисплеях захватывают только 8-битный SDR — 16-битный scRGB-фреймбуфер системного композитора сжимается в SDR, обрезая светлые участки и сужая цветовой охват. Starshot захватывает **исходный HDR-фреймбуфер**:

1. **HDR-захват**: Когда дисплей сообщает о поддержке HDR, запрашивается формат пикселей `R16G16B16A16Float` для получения полных scRGB-данных с плавающей точкой (яркость до тысяч нит)
2. **HDR-сохранение**: 16-битный AVIF / JPEG XL с цветовым охватом BT.2020 + функцией передачи PQ. Без обрезания светлых участков, без сужения охвата
3. **Расчёт maxCLL**: Эффект гистограммы Win2D вычисляет максимальный уровень яркости контента для отличия настоящего HDR-контента от SDR-контента в HDR-формате
4. **Управление цветом**: Чтение ICC-профиля дисплея для анализа реальных первичных цветов охвата и встраивание блоков cICP/ICC в выходные файлы. HDR принудительно BT.2020; SDR можно переключать (вкл. = чтение реального охвата ICC, выкл. = BT.709)

#### Обработка SDR-контента

На HDR-дисплеях рабочий стол и SDR-приложения также захватываются в HDR-формате (R16G16B16A16Float), но фактическая яркость контента находится на уровне SDR. Starshot обрабатывает это следующим образом:

- **По умолчанию**: Сохраняется в HDR-формате (16 бит), **без 8-битной тональной компрессии**, что предотвращает деградацию и искажение цветов
- **Переключатель «Удалять HDR для SDR-контента»** (опционально): При включении определяет порог maxCLL — если контент ниже порога, автоматически преобразует в SDR (используя настроенный пользователем формат сохранения SDR) и удаляет HDR-файл, экономя место

#### Резервный UHDR JPEG

HDR-скриншоты могут также создавать Ultra HDR JPEG (базовое SDR-изображение + HDR gain map), который корректно отображается даже в программах без поддержки HDR. Кодируется через `UhdrEncoder` из `Starward.Codec`.

#### Компромисс HDR при захвате области

Оверлей захвата области **намеренно** выполняет тональную компрессию HDR-кадров в SDR для отображения — потому что `CanvasControl` WinUI использует цепочку обмена SDR, и прямой вывод scRGB с плавающей точкой приводил бы к искажению цветов или черноте. **Сохранённые файлы остаются полноценным HDR**, без изменений; обрезание светлых участков при выборе области влияет только на предпросмотр, но не на результат.

### Три режима захвата

| Режим                    | Цель                                                | Формат буфера обмена      | Файл        |
| ------------------------ | --------------------------------------------------- | ------------------------- | ----------- |
| Полный экран             | Весь дисплей (активное окно / экран с курсором, переключается) | CF_HDROP (файл)           | Сохраняется |
| Область                  | Выделение рамкой / клик по окну                     | CF_DIB (BGRA-битмап)      | Сохраняется |
| Только копирование области| Выделение рамкой / клик по окну                     | CF_DIB (BGRA-битмап)      | Нет         |

Все три режима используют общие: определение HDR, управление цветом, шаблоны имён файлов, конвейер сохранения и информационное всплывающее окно.

### Оверлей захвата области

- **Замороженный кадр**: Сначала все дисплеи объединяются в один битмап; оверлей отображает замороженный кадр — экран не двигается при выборе, а сам оверлей не попадает в скриншот
- **Несколько дисплеев**: Покрывает весь виртуальный экран; лупа и окно координат привязаны к дисплею с курсором (без пересечения экранов)
- **Обнаружение окон**: EnumWindows + фильтрация DWM cloaked/toolwindow + расширенные границы DWM (удаление теней) + двойной кандидат клиентской области + выбор по Z-порядку; клик по окну для прямого захвата (QuickCrop)
- **Лупа**: Целочисленное выравнивание NearestNeighbor + пиксельная сетка (15×15 пикселей, каждый по 10px), пиксели чётко различимы
- **Анимированные бегущие муравьи + координаты в реальном времени**: X/Y/W/H выделения + физические координаты курсора
- **Пиксельная точность**: Выделение рамкой +1px, прямоугольник окна +0
- ESC / Правый клик для отмены, Enter для подтверждения наведения на окно

### Буфер обмена

WinRT `Clipboard.SetContent` в неупакованных WinUI-приложениях ненадёжен (отложенный рендеринг + проблемы с Flush, содержимое часто не доходит до других приложений). Starshot напрямую использует нативные API Win32 (`OpenClipboard` / `SetClipboardData`):

- **Полный экран**: CF_HDROP (формат перетаскивания файлов) — вставьте в Проводник или мессенджер и сразу получите файл
- **Область**: CF_DIB (BGRA-битмап) — SDR-битмап, вырезанный из оверлея, напрямую помещается в буфер обмена, без чтения файла, перекодирования или вторичной тональной компрессии
- Вызывается из любого потока, 10×20 мс повторных попыток при занятом буфере обмена

### Сохранение

- **Плоская структура** (без подпапок), по умолчанию `Изображения\Starshot`, настраивается
- **SDR-формат** (PNG / AVIF / JPEG XL, по умолчанию PNG) и **HDR-формат** (AVIF / JPEG XL, по умолчанию AVIF) настраиваются независимо
- Качество: Среднее / Высокое / Без потерь
- Метаданные XMP (CreatorTool = Starshot)
- Сериализация кодирования (SemaphoreSlim) для предотвращения конфликтов параллельного кодирования
- **Статистика хранилища**: На странице настроек отображается использование дискового пространства скриншотами / кэшем миниатюр / обоями / журналами, с возможностью обновления и очистки кэша в один клик (попутно очищаются осиротевшие файлы обоев)

#### Поддерживаемые форматы

| Формат    | Глубина цвета                    | Поддержка HDR                          | Применение                         |
| --------- | -------------------------------- | -------------------------------------- | ---------------------------------- |
| PNG       | 8 бит / 16 бит                   | Можно сохранить HDR, но плохая совместимость | SDR по умолчанию, без потерь       |
| AVIF      | 8 бит / 10 бит / 12 бит          | Полный HDR                             | HDR по умолчанию, высокое сжатие   |
| JPEG XL   | 8 бит / 16 бит                   | Полный HDR                             | Альтернатива HDR, обратимое сжатие |
| UHDR JPEG | 8 бит + gain map                 | SDR-совместимый HDR-резерв             | Дополнительный HDR-вывод           |

### Шаблоны имён файлов

Для полного экрана и области используются **независимые шаблоны**.

| Подстановка                                                | Значение                                     | Пример              |
| ---------------------------------------------------------- | -------------------------------------------- | ------------------- |
| `{process}`                                                | Имя процесса (без расширения)                | `explorer`          |
| `{processPath}`                                            | Имя exe-файла (с расширением)                | `explorer.exe`      |
| `{title}`                                                  | Заголовок окна (trim + настраиваемая длина обрезки) | `Genshin Impact` |
| `{timestamp}`                                              | Unix-временная метка                         | `1721234567`        |
| `{time}`                                                   | yyyyMMdd_HHmmssff                            | `20260718_14302512` |
| `{date}`                                                   | yyyyMMdd                                     | `20260718`          |
| `{width}` `{height}`                                       | Размеры изображения (px)                     | `1920` `1080`       |
| `{year}` `{month}` `{day}` `{hour}` `{minute}` `{second}`  | Отдельные компоненты времени                 |                     |

Недопустимые символы в именах файлов единообразно заменяются на `_`.

### Информационное всплывающее окно

После скриншота появляется миниатюра + статусное окно (не влияет на скриншоты — установлен `WDA_EXCLUDEFROMCAPTURE`, другие инструменты захвата не видят это окно):

- **Обработка** (анимация вращения) / **Сохранено** (с кнопкой открытия) / **Скопировано** (зелёная галочка) / **Ошибка**
- Счётчик последовательных снимков (например, 2/3)
- Composition-анимация выезда/заезда

### Библиотека скриншотов

- Навигация по нескольким папкам (папка скриншотов по умолчанию + добавленные пользователем папки)
- `FileSystemWatcher` обнаруживает добавления/удаления в реальном времени
- Группировка по дате, ленивая загрузка миниатюр
- Контекстное меню: Открыть / Копировать файл / Копировать как JPG / Открыть в Проводнике / Открыть с помощью / Удалить
- Множественный выбор + перетаскивание + вход в пакетное преобразование

### Просмотрщик изображений

- Масштабирование (ползунок / кнопки / колёсико мыши / двойной клик для подгонки), полноэкранный режим (F11)
- Предыдущее / Следующее (клавиши стрелок, колёсико мыши, нижняя полоса миниатюр)
- Перетаскивание файлов для открытия
- Контекстное меню: Копировать файл / путь / изображение, Удалить, Открыть в Проводнике, Открыть с помощью
- **Панель редактирования**: Переключение режима отображения HDR / SDR / Auto, ползунок яркости SDR (100–500 нит), информация об изображении и дисплее
- **Преобразование формата**: Экспорт в PNG / AVIF / JPEG XL (SDR-дисплей) или UHDR JPEG / AVIF / JPEG XL (HDR-дисплей)
- **Управление цветом**: Чтение ICC-профиля дисплея и AdvancedColorInfo

### Пакетное преобразование форматов

| Направление преобразования            | Движок                                  |
| ------------------------------------- | --------------------------------------- |
| JPG / PNG → AVIF / JXL                | avifenc.exe / cjxl.exe (CLI)            |
| AVIF / JXL → JPG / PNG                | avifdec.exe / djxl.exe (CLI)            |
| JXR / WEBP / HEIC и др. → AVIF / JXL  | Внутрипроцессный ImageSaver (avifEncoderLite) |

### Персонализация

- **Пользовательские обои**: Три режима
  - **Фиксированное изображение**: Выберите изображение, отображается постоянно
  - **Фиксированное видео**: Цикличное беззвучное воспроизведение, автоматическая пауза при скрытии главного окна
  - **Случайно из папки**: При каждом запуске случайно выбирается изображение или видео из папки
  - Автоматическое обнаружение пропавших источников обоев, очистка конфигурации и возврат к отсутствию обоев + уведомление toast
- **Цвет акцента**:
  - **Автоподбор из обоев** (включено по умолчанию): Определение доминирующего цвета обоев как акцентного цвета приложения (усиление насыщенности HSV); для видео — только первый кадр, чтобы избежать мерцания цвета
  - **Пользовательский цвет**: Ручной выбор цвета переопределяет автоподбор
- **Тема**: Следовать системной / Светлая / Тёмная
- **Эффект акрила**: В режиме обоев можно выбрать слой матового стекла или прямую прозрачность обоев

### Экран-заставка

При запуске отображается логотип + слоган с задержкой 700 мс и последующим затуханием 400 мс. Срабатывает только при первом открытии окна; не воспроизводится при восстановлении из системного трея.

### Системный трей

- Левый клик показывает главное окно, правый клик — контекстное меню (Показать / Выход)
- Закрытие главного окна сворачивает в трей (можно отключить)
- Механизм `ForceExit` гарантирует, что «Выход» из трея действительно завершает программу

### Автозапуск

- Реестр `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`, указывает на лаунчер (корневой `Starshot.exe`)
- Опциональный флаг `--hide` для запуска свёрнутым в трей (требуется включённый трей)

## Архитектура

### Структура каталогов

```
Корень/
  Starshot.exe            ← C++ лаунчер (~400 КБ, запускает основную программу из app/)
  StarshotDatabase.db     ← База данных настроек SQLite
  app/
    Starshot.exe          ← Основная программа (WinUI 3 / .NET 10)
    *.dll                 ← Зависимости
    avifenc.exe и др.     ← Инструменты кодеков (из Starward.Codec NuGet)
%LOCALAPPDATA%/Starshot/  (по умолчанию, настраивается)
  log/                    ← Журналы
  cache/                  ← Кэш миниатюр
```

### Лаунчер

Нативная C++ программа (~400 КБ), в настоящее время жёстко привязана к `app/Starshot.exe`. В будущем планируется поддержка `version.ini` для версионированных каталогов + автоматическая очистка старых версий.

### Технологический стек

| Уровень             | Технология                                                               |
| ------------------- | ------------------------------------------------------------------------ |
| UI-фреймворк        | WinUI 3 (Windows App SDK 1.8)                                            |
| Среда выполнения    | .NET 10                                                                  |
| Графика             | Win2D 1.3 (взаимодействие с D3D11, тональная компрессия HDR, эффект гистограммы) |
| Кодеки              | Starward.Codec NuGet (P/Invoke-обёртка libavif / libjxl / UltraHDR)      |
| Хранение данных     | SQLite + Dapper                                                          |
| Журналирование      | Serilog                                                                  |
| Системный трей      | H.NotifyIcon.WinUI                                                       |
| Миниатюры           | Scighost.WinUI ImageEx + пользовательский CachedImage                    |
| Оверлей области     | Win2D CanvasControl (рендер замороженного кадра + отрисовка выделения)   |
| Буфер обмена        | Нативное API Win32 (OpenClipboard / SetClipboardData)                    |
| Лаунчер             | Нативный C++ (инструментарий v145, статический CRT)                      |

### Защита от повторного входа

Глобальный guard `Interlocked.CompareExchange` — режимы полного экрана, области и только копирования используют общий флаг `_isCapturing`, предотвращая множественные захваты при быстрых нажатиях клавиш или последовательных нажатиях горячих клавиш.

### Конфигурация сборки


## Известные ограничения

- Оверлей захвата области отображает HDR-кадры как SDR (WinUI CanvasControl использует цепочку обмена SDR); сохранённые файлы не затрагиваются
- Пользовательские обои используют заполнение `UniformToFill`, но обрезка WinUI не центрирована — в настоящее время выравнивание **по верхнему левому краю**, например, узкие (вертикальные) обои в широком окне будут показывать только верхнюю часть (обрезается сверху, а не по центру)
- При открытии оверлея захвата области курсор остаётся в системной форме по умолчанию; **переместите мышь один раз**, чтобы появился курсор-перекрестие (WinUI `ProtectedCursor` не применяется немедленно к неподвижному указателю, уже находящемуся на элементе — одно перемещение запускает событие указателя, после чего всё работает нормально)
- Пока нет управления версиями / автообновления
