## Зачем нужен Starshot

Встроенные инструменты создания скриншотов Windows (Snipping Tool, Win+Shift+S) даже на HDR-дисплеях выдают только 8-битные SDR-изображения — системный композитор сжимает 16-битные HDR-кадры, обрезая яркие участки и сужая цветовой охват. Распространённые сторонние инструменты (ShareX и др.) также ограничены традиционным конвейером GDI/BitBlt и не могут работать с HDR-данными.

Starshot напрямую захватывает исходный `R16G16B16A16Float` scRGB-фреймбуфер с уровня DXGI, полностью сохраняя информацию о яркости HDR (до нескольких тысяч нит). Скриншоты кодируются в 16-битный HDR AVIF или JPEG XL с метаданными цветового пространства BT.2020 и передаточной функции PQ. Также доступны автоматическая деградация для SDR-дисплеев, снимок области, пакетное преобразование форматов и всё, что ожидается от универсального инструмента для скриншотов.

**Ключевые особенности**

- 🎯 **Полный HDR-конвейер без потерь** — захват, кодирование и управление цветом полностью в 16 битах. Никакого тон-маппинга с потерями.
- 🧠 **Умное определение HDR/SDR** — гистограмма maxCLL отличает настоящий HDR-контент от SDR-контента в HDR-обёртке.
- ✂️ **Снимок области** — оверлей с замороженным кадром на нескольких мониторах, определение окон и лупа для попиксельно точного выделения.
- 📋 **Нативный буфер обмена** — прямая запись через Win32 API, без проблем отложенного рендеринга WinRT.
- 🗂️ **Поддержка множества форматов** — AVIF / JPEG XL / UHDR JPEG / PNG, включая пакетное преобразование.
- 📦 **Портативный** — распакуйте и запускайте. Без установки, без прав администратора.

## Возможности

### Конвейер HDR-скриншотов

Большинство инструментов даже на HDR-дисплеях захватывают только 8-битный SDR — 16-битный scRGB-кадр системного композитора сжимается в SDR с обрезанием светлых участков и сужением цветового охвата. Starshot захватывает **исходный HDR-фреймбуфер**:

1. **HDR-захват**: когда дисплей сообщает о режиме HDR, запрашивается формат пикселей `R16G16B16A16Float` для получения полных scRGB-данных с плавающей точкой (яркость до нескольких тысяч нит)
2. **HDR-сохранение**: 16-битный AVIF / JPEG XL, цветовое пространство BT.2020 + передаточная функция PQ. Яркие участки не обрезаются, цветовой охват не сужается
3. **Расчёт maxCLL**: эффект гистограммы Win2D вычисляет максимальную яркость контента для различения настоящего HDR и SDR в HDR-контейнере
4. **Управление цветом**: чтение ICC-профиля дисплея для извлечения реальных первичных цветов охвата, запись чанков cICP/ICC в файл. Для HDR принудительно BT.2020; для SDR управление цветом можно включить/отключить (вкл = чтение реального охвата из ICC, выкл = BT.709)

#### Обработка SDR-контента

На HDR-дисплее рабочий стол и SDR-приложения также захватываются в HDR-формате (R16G16B16A16Float), но реальная яркость контента — на уровне SDR. Starshot обрабатывает это следующим образом:

- **По умолчанию**: сохраняется в HDR-формате (16 бит), **без 8-битного тон-маппинга**, чтобы избежать деградации и цветовых сдвигов
- **Переключатель «Удалять HDR для SDR-контента»** (опционально): при включении, если контент ниже порога maxCLL, автоматически преобразует в SDR (в соответствии с пользовательским форматом сохранения SDR) и удаляет HDR-файл для экономии места

#### Резервный UHDR JPEG

HDR-скриншоты могут одновременно создавать Ultra HDR JPEG (SDR-основа + HDR gain map), который корректно отображается даже в программах без поддержки HDR. Кодируется через `UhdrEncoder` из `Starward.Codec`.

#### Компромисс HDR при снимке области

Оверлей снимка области **намеренно** тон-маппит HDR-кадры в SDR для отображения — потому что `CanvasControl` в WinUI использует SDR-цепочку обмена, и прямой вывод scRGB с плавающей точкой вызывает цветовые искажения или затемнение. **Сохранённый файл — полноценный HDR**, нетронутый; сжатие ярких участков при выделении влияет только на предпросмотр, но не на результат.

### Три режима скриншотов

| Режим                  | Цель                                                     | Формат буфера обмена      | Файл         |
| ---------------------- | -------------------------------------------------------- | ------------------------- | ------------ |
| Весь экран             | Весь монитор (активное окно / экран с курсором, переключаемый) | CF_HDROP (файл)           | Сохраняется  |
| Область                | Прямоугольное выделение / клик по окну                   | CF_DIB (BGRA-растр)       | Сохраняется  |
| Копирование области    | Прямоугольное выделение / клик по окну                   | CF_DIB (BGRA-растр)       | Нет          |

Все три режима используют общие HDR-обнаружение, управление цветом, шаблоны имён файлов, конвейер сохранения и информационное уведомление.

### Оверлей снимка области

- **Замороженный кадр**: сначала все мониторы собираются в одно растровое изображение; оверлей отображает этот замороженный кадр, чтобы изображение оставалось неподвижным при выделении. Сам оверлей исключается из скриншота.
- **Несколько мониторов**: охватывает весь виртуальный экран. Лупа и окно координат ограничены монитором с курсором (без перехода через границы экранов).
- **Определение окон**: EnumWindows + фильтрация DWM cloaked/toolwindow + расширенные границы DWM (удаление теней) + двойной кандидат клиентской области + выбор по Z-порядку. Клик по окну для прямого захвата (QuickCrop).
- **Лупа**: целочисленное выравнивание NearestNeighbor + пиксельная сетка (15×15 пикселей, каждый по 10px), пиксели чётко различимы.
- **Анимированная выделительная рамка + координаты в реальном времени**: X/Y/W/H выделения + физические координаты курсора.
- **Попиксельная точность**: перетаскивание выделения +1px; прямоугольник окна +0.
- Esc / правый клик для отмены; Enter для подтверждения захвата окна под курсором.

### Буфер обмена

WinRT `Clipboard.SetContent` в неупакованных WinUI-приложениях ненадёжен (отложенный рендеринг + проблемы с Flush, контент часто не доходит до других приложений). Starshot использует нативные Win32 API (`OpenClipboard` / `SetClipboardData`) напрямую:

- **Скриншот всего экрана**: CF_HDROP (формат перетаскивания файла) — вставьте в Проводник или мессенджер, чтобы сразу получить файл
- **Снимок области**: CF_DIB (BGRA-растр) — вырезанный SDR-растр из оверлея помещается прямо в буфер обмена, без чтения файла, без перекодирования, без вторичного тон-маппинга
- Можно вызывать из любого потока, с 10×20 мс повторными попытками при занятом буфере обмена

### Сохранение

- **Плоская структура** (без подпапок). По умолчанию `Изображения\Starshot`, настраивается.
- **Формат SDR** (PNG / AVIF / JPEG XL; по умолчанию PNG) и **формат HDR** (AVIF / JPEG XL; по умолчанию AVIF) настраиваются раздельно.
- Уровни качества: Среднее / Высокое / Без потерь.
- Метаданные XMP (CreatorTool = Starshot).
- Сериализованное кодирование (SemaphoreSlim) для предотвращения конфликтов параллельного кодирования.
- **Статистика хранилища**: на странице настроек отображается занятое место для скриншотов / кэша миниатюр / обоев / журналов / резервных копий, с возможностью обновления и очистки кэша в один клик (попутно удаляются осиротевшие файлы обоев).

#### Поддерживаемые форматы

| Формат     | Глубина цвета               | Поддержка HDR                    | Применение                         |
| ---------- | --------------------------- | -------------------------------- | ---------------------------------- |
| PNG        | 8 бит / 16 бит              | Можно сохранить, но плохая совместимость | SDR по умолчанию, без потерь       |
| AVIF       | 8 бит / 10 бит / 12 бит     | Полный HDR                       | HDR по умолчанию, высокое сжатие   |
| JPEG XL    | 8 бит / 16 бит              | Полный HDR                       | Альтернатива HDR, обратимое сжатие |
| UHDR JPEG  | 8 бит + gain map            | SDR-совместимый HDR-фолбэк       | Дополнительный вывод HDR           |

### Шаблоны имён файлов

Для скриншотов всего экрана и области используются **независимые шаблоны**.

| Плейсхолдер                                                 | Значение                                       | Пример               |
| ----------------------------------------------------------- | ---------------------------------------------- | -------------------- |
| `{process}`                                                 | Имя процесса (без расширения)                  | `explorer`           |
| `{processPath}`                                             | Имя exe-файла (с расширением)                  | `explorer.exe`       |
| `{title}`                                                   | Заголовок окна (trim + настраиваемая длина обрезки) | `Genshin Impact`     |
| `{timestamp}`                                               | Unix-временная метка                           | `1721234567`         |
| `{time}`                                                    | yyyyMMdd_HHmmssff                              | `20260718_14302512`  |
| `{date}`                                                    | yyyyMMdd                                       | `20260718`           |
| `{width}` `{height}`                                        | Размеры изображения (px)                       | `1920` `1080`        |
| `{year}` `{month}` `{day}` `{hour}` `{minute}` `{second}`   | Компоненты времени                             |                      |

Недопустимые символы в именах файлов единообразно заменяются на `_`.

### Информационное уведомление

После скриншота появляется миниатюра + статусное уведомление (не влияет на скриншоты — установлен флаг `WDA_EXCLUDEFROMCAPTURE`, другие инструменты не могут захватить это окно):

- **Обработка** (анимация вращения) / **Сохранено** (с кнопкой «Открыть») / **Скопировано** (зелёная галочка) / **Ошибка**
- Счётчик серийных снимков (например, 2/3)
- Анимации скольжения Composition (вход/выход)

### Библиотека скриншотов

- Навигация по нескольким папкам (каталог скриншотов по умолчанию + добавленные пользователем папки)
- `FileSystemWatcher` для отслеживания добавлений/удалений в реальном времени
- Группировка по дате, ленивая загрузка миниатюр
- Контекстное меню: Открыть / Копировать файл / Копировать как JPG / Открыть в Проводнике / Открыть с помощью / Удалить
- Множественный выбор + перетаскивание + вход в пакетное преобразование

### Просмотр изображений

- Масштабирование (ползунок / кнопки / колёсико мыши / двойной клик для подгонки), полноэкранный режим (F11)
- Предыдущее / Следующее (клавиши стрелок, колёсико мыши, полоса миниатюр внизу)
- Перетащите файл для прямого открытия
- Контекстное меню: Копировать файл / путь / изображение, Удалить, Открыть в Проводнике, Открыть с помощью
- **Панель редактирования**: переключение режимов отображения HDR / SDR / Auto, ползунок яркости SDR (100–500 нит), информация об изображении и дисплее
- **Преобразование форматов**: экспорт в PNG / AVIF / JPEG XL (SDR-дисплей) или UHDR JPEG / AVIF / JPEG XL (HDR-дисплей)
- **Управление цветом**: чтение ICC-профиля дисплея и AdvancedColorInfo

### Пакетное преобразование форматов

| Направление                          | Движок                                |
| ------------------------------------ | ------------------------------------- |
| JPG / PNG → AVIF / JXL               | avifenc.exe / cjxl.exe (CLI)          |
| AVIF / JXL → JPG / PNG               | avifdec.exe / djxl.exe (CLI)          |
| JXR / WEBP / HEIC и др. → AVIF / JXL | Внутрипроцессный ImageSaver (avifEncoderLite) |

### Персонализация

- **Пользовательские обои**: три режима
  - **Определённое изображение**: выбрать картинку, показывается постоянно
  - **Определённое видео**: цикличное беззвучное воспроизведение; автоматическая пауза при скрытии главного окна
  - **Случайное из папки**: при каждом запуске выбирается случайное изображение или видео из папки
  - Автоопределение потери источника обоев, очистка конфигурации и возврат к отсутствию обоев + уведомление
- **Акцентный цвет**:
  - **Автовыбор из обоев** (включено по умолчанию): доминирующий цвет обоев используется как акцентный (с усилением насыщенности HSV); для видео сэмплируется только первый кадр во избежание мерцания
  - **Пользовательский цвет**: ручной выбор цвета переопределяет автовыбор
- **Тема**: Следовать системе / Светлая / Тёмная
- **Эффект акрила**: в режиме обоев можно выбрать матовое стекло или прямую прозрачность обоев

### Заставка при запуске

Отображает логотип + слоган при запуске. Задержка 700 мс, затем исчезновение за 400 мс. Срабатывает только при первом открытии окна; не повторяется при восстановлении из системного трея.

### Системный трей

- Левый клик показывает главное окно, правый клик открывает контекстное меню (Показать / Выход)
- Закрытие главного окна сворачивает в трей (можно отключить)
- Механизм `ForceExit` гарантирует, что «Выход» из трея действительно завершает приложение

### Автозапуск

- Ключ реестра `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`, указывающий на лаунчер (корневой `Starshot.exe`)
- Опция `--hide` для запуска свёрнутым в трей (требуется включённый трей)
- Переключатель читает реестр в реальном времени (без кэширования в БД): отключение через Диспетчер задач меняет только StartupApproved, не удаляя запись Run — переключатель всё ещё показывает «Вкл»
- При запуске проверяет существование exe, на который указывает запись автозапуска; если файла нет, автоматически удаляет запись и показывает уведомление

### Проверка обновлений

- Проверка при запуске с ограничением по времени (≥24 ч + переключатель включён) последней версии на GitHub Releases, либо ручная проверка со страницы «О программе»
- Обновления используют потоковую распаковку SharpCompress (прямое сетевое подключение, без сохранения zip на диск). Каждая запись пишется прямо в корневой каталог. При ошибке — восстановление предыдущего состояния. При успехе — перезапуск лаунчера с `--clean` для удаления старых версий
- Проверяются только CI/CD-релизы (читается номер версии из `version.ini`). Локальные сборки (без `version.ini`, `AppVersion = Local`) считаются как 0.0.0 и могут обновиться до любого CI/CD-релиза
- Соглашение о регистре версии: тег GitHub, имя zip и каталог `app-{version}/` — в нижнем регистре (напр. `0.3.1-preview`); `version.ini` сохраняет исходный регистр (`0.3.1-Preview`, отображается на странице «О программе»), лаунчер приводит его к нижнему регистру при поиске каталога.

```
  version.ini             ← Номер версии (только в CI/CD-релизах; отсутствует в локальных сборках)
  app-{version}/          ← Каталог основной программы (версионирован для CI/CD-релизов, app/ для локальных сборок)
    Starshot.exe          ← Основная программа (WinUI 3 / .NET 10)
    *.dll                 ← Зависимости
    avifenc.exe и др.     ← Инструменты кодеков (из Starward.Codec NuGet)
  backup/                 ← Резервные копии БД
%LOCALAPPDATA%/Starshot/  (по умолчанию, настраивается)
  log/                    ← Журналы
  bg/                     ← Обои
  thumb/                  ← Кэш миниатюр
```

### Лаунчер

Нативная программа на C++ (~400 КБ). Читает `version.ini` для определения, какой `app-{version}/Starshot.exe` запускать (если version.ini отсутствует — `app/`, debug/local сборка). При запуске с `--clean` (или `--clean=<pid>`) обходит каталоги `app-*` и удаляет не соответствующие текущей версии — сначала 10 быстрых попыток; с pid продолжает повторять раз в минуту до 5 мин, затем принудительно завершает старый главный процесс по pid перед последним удалением (без pid — отказ после 10 попыток).

### Трей и фоновый запуск

- `--hide`: при автозапуске MainWindow не создаётся. Глобальные горячие клавиши регистрируются на hwnd SystemTrayWindow (окно трея служит постоянным хостом)
- TaskbarIcon из H.NotifyIcon.WinUI требует одного Window.Show для вызова `Loaded` перед регистрацией иконки. При инициализации `WS_EX_LAYERED + alpha=0` делает окно прозрачным для выполнения этого Show, избегая видимой вспышки при автозапуске с `--hide`
- C++ лаунчер пересобирает `argv[1..]` для прозрачной передачи аргументов командной строки

### Защита от повторного входа

Глобальный страж `Interlocked.CompareExchange`. Режимы всего экрана, области и только копирования разделяют один флаг `_isCapturing` — повторные нажатия клавиш или быстрое последовательное срабатывание горячих клавиш не вызовут множественных захватов.

## Известные ограничения

- Оверлей снимка области отображает HDR-кадры как SDR (CanvasControl WinUI использует SDR-цепочку обмена); сохранённые файлы не затрагиваются
- Пользовательские обои используют `UniformToFill` для заполнения окна, но обрезка WinUI не центрируется — в настоящее время выравнивание **по левому верхнему углу**. Например, узкие (вертикальные) обои в широком окне покажут только верхнюю часть (обрезается сверху, а не по центру)
- При открытии оверлея снимка области курсор остаётся в системной форме по умолчанию. **Нужно один раз подвинуть мышь**, чтобы появился курсор-перекрестие (`ProtectedCursor` WinUI не срабатывает мгновенно на неподвижном указателе, уже находящемся над элементом; движение запускает событие pointer, после чего всё работает нормально)
- На двух мониторах с разным DPI (напр. основной 150 %, дополнительный 125 %) координаты оверлея захвата области на дополнительном мониторе сбиты (лупа / выделение не совпадают). Обход: установить одинаковый масштаб на обоих мониторах

## Часто задаваемые вопросы

<details>
<summary><b>В библиотеке скриншотов (главная страница) цвета изображений некорректные / искажённые</b></summary>

Обычно это проблема системных кодеков изображений Windows (расширения AVIF / HEIF / JPEG XL), а не баг Starshot. Попробуйте найти и обновить следующие компоненты в Microsoft Store:

- **AV1 Video Extension**
- **HEIF Image Extensions**
- **HEVC Video Extensions**
- **Webp Image Extensions**

После обновления перезапустите Starshot. Если проблема сохраняется, [создайте Issue](../../../issues/new) с приложенным скриншотом.

</details>

<details>
<summary><b>Цвета скриншота отличаются от того, что я вижу на экране</b></summary>

Если вы используете HDR-дисплей, убедитесь, что переключатель HDR в Windows включён (Параметры → Система → Дисплей → HDR). Функция HDR-скриншотов работает только в режиме HDR.

</details>

<details>
<summary><b>Не удаётся вставить из буфера обмена после скриншота</b></summary>

Starshot использует нативный Win32 API для записи в буфер обмена, что теоретически надёжнее, чем WinRT. Если вставка всё равно не удаётся, возможно, целевое приложение не поддерживает соответствующий формат (CF_HDROP для файлов / CF_DIB для растров). Попробуйте вставить в Проводник (файлы) или Paint (растр) для проверки.

</details>
